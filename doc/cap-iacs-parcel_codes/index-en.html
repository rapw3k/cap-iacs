Abstract

This ontology is one of the modules comprising CAP-IACS (Common Agriculture Policy - Integrated Administration and Control System) ontology. The CAP-IACS ontology was developed part of the Open-IACS project, which aims to provide an open platform for the exchange of solutions in the field of Integrated Agricultural Control System (IACS) through the paradigm of connected open data. The ontology has been derived from the derived from the JRC IACS application schema.

Classification correctnessni back to ToC or Named Individual ToC

IRI: http://lpis.ec.europa.eu/registry/applicationschema/cap-iacs-parcel/code#DataQualityMeasureTypeIuiCcValue/classificationCorrectness

--Name-- Classification correctness -- Definition -- Thematic classification correctness as defined in ISO 19157:2013. -- Description -- Thematic classification correctness applied to the land cover class. EXAMPLE: A MS has defined Hedge landscape feature and Line of trees landscape feature. Attributing an instance of hedge to line of trees class is a classification error. NOTE 1. The correct classification and quantification of the land cover types (arable/PC) is directly used for assessing the exemption for the green payments (EFA, crop diversification) NOTE 2. The correct classification of the grassland is relevant because it acts as a threshold fors crop diversification. Certain types of grassland (pro-rata or not) are subject to specific requirements and therefore need to be distinguished (thus recorded). In general, such detailed inspection triggers a better understanding of the pro-rata behaviour in the LPIS. NOTE 3. Each pro-rata type shall be regarded as a separate land cover class For example permanent grassland of 50% is one class, and permanent grassland 90% is another class. Therefore classifying an instance of permanent grassland of 50% to class permanent grassland 90% a is a classification error (like in the example above with the hedge and line of trees).
